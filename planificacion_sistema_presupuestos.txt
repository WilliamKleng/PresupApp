SISTEMA DE PRESUPUESTOS Y REMITOS — PLAN DE DESARROLLO
=========================================================

DESCRIPCIÓN DE LA APLICACIÓN
------------------------------
Aplicación web de uso personal para la gestión de presupuestos y
remitos de servicios/trabajos. Reemplaza el flujo manual en Excel
por un sistema centralizado accesible desde el navegador.

El sistema permite:
  - Gestionar una cartera de clientes (alta, edición, eliminación)
  - Crear presupuestos detallados con ítems de texto libre,
    cantidades, precios unitarios y opcionalmente IVA
  - Crear remitos vinculados a clientes con ítems sin precio
  - Numeración autoincremental y formateada (0001, 0002...) para
    presupuestos y remitos de forma independiente
  - Visualizar cada documento con un formato fiel al template
    original en Excel, listo para imprimir o exportar a PDF
  - Acceder a un panel resumen con los últimos movimientos

El acceso está protegido por login de usuario único (uso personal).
No requiere internet ni servidor externo: corre localmente.



STACK
-----
Backend:       Flask + Flask-Login
Base de datos: SQLite via SQLAlchemy
Frontend:      Jinja2 + CSS propio (design system del kiosk)
PDF/Impresión: WeasyPrint (genera PDF server-side)


BASE DE DATOS — ESQUEMA
------------------------

clientes
  id | nombre | direccion | telefono | mail | created_at

presupuestos
  id | numero (autoincremental) | cliente_id | fecha |
     iva_porcentaje (nullable) | notas | created_at

presupuesto_items
  id | presupuesto_id | orden | concepto | cantidad |
     precio_unitario | total

remitos
  id | numero (autoincremental) | cliente_id | fecha | notas

remito_items
  id | remito_id | orden | concepto | cantidad

NOTA: El número autoincremental NO es el ID de SQLite. Es un campo
separado calculado como MAX(numero) + 1 al crear, formateado como
0001, 0002, etc. Así si se elimina un registro, la numeración no
queda con huecos visibles ni se rompe.


MÓDULOS Y PANTALLAS
--------------------

MÓDULO 1 — CLIENTES
  - Listado con búsqueda
  - Formulario alta/edición (nombre, dirección, teléfono, mail)
  - Eliminación con confirmación
  - Al eliminar cliente con presupuestos asociados: advertencia
    (no borrar en cascada)

MÓDULO 2 — PRESUPUESTOS
  - Listado: número, cliente, fecha, total — con filtro por cliente
  - Formulario de creación/edición:
      · Selector de cliente (con datos del cliente al lado)
      · Fecha (por defecto hoy)
      · Tabla de ítems dinámica: agregar/quitar filas, concepto libre,
        cantidad, precio unitario → total por fila calculado en JS
      · Toggle IVA: si está activado, ingresás el porcentaje y se
        muestra subtotal + IVA + total
      · Total general
  - Vista previa (renderiza el template igual al Excel)
  - Botón imprimir (window.print() con CSS de impresión)
  - Exportar PDF (WeasyPrint server-side, descarga el .pdf)
  - Eliminar con confirmación

MÓDULO 3 — REMITOS
  - Igual que presupuestos pero:
      · Numeración propia e independiente
      · Sin columna precio ni total
      · Campos propios si se necesitan (ej: condición de entrega)

MÓDULO 4 — DASHBOARD / HOME
  - Últimos 5 presupuestos creados
  - Últimos 5 remitos
  - Contador total: clientes, presupuestos, remitos
  - Acceso rápido a "Nuevo presupuesto" / "Nuevo remito"


FASES DE DESARROLLO
--------------------

FASE 1 — Base del proyecto (~2-3hs)
  - Setup Flask, SQLAlchemy, estructura de carpetas
  - Modelos de DB + migraciones iniciales
  - Login básico (Flask-Login, un solo usuario en config)
  - Layout base con sidebar (design system del kiosk)

FASE 2 — Clientes (~2hs)
  - CRUD completo
  - Validaciones básicas (mail, campos obligatorios)

FASE 3 — Presupuestos (~4-5hs)
  - Listado + creación con ítems dinámicos (JS)
  - Cálculo automático de totales en el form
  - Vista previa fiel al template Excel
  - Impresión web (CSS @media print)
  - Export PDF con WeasyPrint

FASE 4 — Remitos (~2-3hs)
  - Misma lógica que presupuestos, simplificada

FASE 5 — Dashboard y pulido (~1-2hs)
  - Home con resumen
  - Búsqueda/filtros en listados
  - Mensajes de éxito/error (flash messages)
